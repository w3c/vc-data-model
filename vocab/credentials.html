This document describes the Verifiable Claims Vocabulary and Term definitions used for describing Verfiable Claims [[verifiable-claims-data-model]]. This document provides the RDFS [[RDF-SCHEMA]] vocabulary definition and a description of the JSON-LD context definition for use with defining Verfiable Claims.

Alternate versions of the vocabulary definition exist in Turtle and JSON-LD, which also includes the @context required for metadata descriptions.

Introduction

This document describes the RDFS vocabulary description used for Verifiable Claims [[verifiable-claims-data-model]] along with the default JSON-LD Context.

VerifiableProfile Verifiable Profile

A `verifiable profile` is a profile that is tamper-resistant and whose contents are typically counter-signed by the `holder` or `subject`.

Property Definitions

The following are property definitions in the shex namespace:

claim claim

A statement made about a `subject`.

rdfs:domain
cred:Credential
credential credential

A set of one or more claims made by the same `entity` about a `subject`. A **verifiable credential** is a credential that is tamper-resistant and whose authorship can be cryptographically verified.

rdfs:range
cred:Credential
rdfs:domain
cred:VerifiableProfile
credentialStatus credential status

The value of this property MUST be a status scheme that provides enough information to determine the current status of the credential (e.g. suspended, revoked, etc.). The status scheme will vary depending on a variety of factors, such as whether it is simple to implement or privacy-enhancing.

rdfs:domain
cred:Credential
entity entity

A thing with distinct and independent existence such as a person, organization, concept, or device.
evidence evidence

The value of this property MUST be one or more evidence schemes that provides enough information to a verifier to determine whether or not the evidence gathered meets their requirements.

rdfs:range
cred:DocumentationVerifier
rdfs:domain
cred:Credential
expires expires

The value of this property MUST be a string value of an [[ISO8601]] combined date and time string and represents the date and time the `credential` will cease to be valid.

rdfs:range
xsd:dateTime
rdfs:domain
cred:Credential
issued issued

The value of this property MUST be a string value of an [[ISO8601]] combined date and time string and represents the date and time the `credential` was issued. Note that this date represents the earliest date when the information associated with the _claim property became valid.

rdfs:range
xsd:dateTime
rdfs:domain
cred:Credential
issuer issuer

The value of this property MUST be a URI. It is RECOMMENDED that dereferencing the URI results in a document containing machine-readable information about the issuer that may be used to verify the information expressed in the `credential`.

rdfs:domain
cred:Credential
proof proof

The method used for a mathematical proof will vary by representation language and the technology used. For example, if digital signatures are used for the proof mechanism, this property is expected to have a value that is a set of name-value pairs including at least a signature, a reference to the signing entity, and a representation of the signing date.

rdfs:domain
Union of cred:Credential cred:VerifiableProfile
revocation revocation

rdfs:domain
cred:Credential
termsOfUse terms of use

The value of this property MUST be one or more terms of use descriptions that provide enough information to a verifier to determine how they may utilize the given information.

rdfs:range
cred:Policy
rdfs:domain
cred:Credential
verifiableCredential verifiable credential

The contents of the `verifiableCredential` property are `verifiable credentials` as described by this specification.

rdfs:range
cred:Credential
rdfs:domain
cred:VerifiableProfile
